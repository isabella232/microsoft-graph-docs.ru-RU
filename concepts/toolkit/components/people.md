# <a name="people-component-in-the-microsoft-graph-toolkit"></a>Компонент "люди" в наборе инструментов Microsoft Graph

Вы можете использовать `mgt-people` веб-компонент для отображения группы людей или контактов с помощью фотографий или инициалов. По умолчанию отображаются наиболее часто используемые контакты для пользователя, выполнившего вход в систему.

Этот компонент использует несколько элементов управления " [центр](./person.md) управления", но их можно привязать к набору дескрипторов людей. Если количество людей для отображения больше `show-max` , чем значение, будет добавлено число, указывающее количество дополнительных контактов.

## <a name="properties"></a>Свойства

По умолчанию `mgt-people` компонент получает события от `/me/people` конечной точки с `personType/class eq 'Person'` фильтром для отображения часто обращенных пользователей. Для изменения этого поведения можно использовать несколько свойств.

| Атрибут | Свойство | Описание |
| --- | --- | --- |
| Show — Max | шовмакс | Указывает максимальное количество людей для отображения. Значение по умолчанию — 3. |
| people | people | Массив пользователей, который получает или задает список людей, отображаемых компонентом. Используйте это свойство для доступа к пользователям, загруженным компонентом. Присвойте этому параметру значение загрузка собственных пользователей. |
| Идентификатор группы | groupId | Извлекает пользователей из определенного идентификатора Microsoft Graph. |
| идентификаторы пользователей | userIds | При наличии массива пользователей Microsoft Graph `ids` компонент будет отображать этих пользователей.  |
| люди — запросы | пеоплекуериес | При наличии массива запросов Person (Names, UPN, сообщений электронной почты) компонент будет отображать этих пользователей. |
| Person — карточка | персонкард | Перечисление для определения действия пользователя, необходимого для активации всплывающей панели `hover` `click` . Значение по умолчанию: `none`. |
| Показать — присутствие | шовпресенце | Логическое значение, определяющее, следует ли отображать значок присутствия пользователя на изображении пользователя. |


В приведенном ниже примере задается максимальное число людей для отображения.

```html
<mgt-people
  show-max="4">
</mgt-people>
```

## <a name="css-custom-properties"></a>Настраиваемые свойства CSS

`mgt-people`Компонент определяет следующие настраиваемые свойства CSS.

```css
mgt-people {
  --list-margin: 8px 4px 8px 8px; /* Margin for component */
  --avatar-margin: 0 4px 0 0; /* Margin for each person */
}
```

## <a name="templates"></a>Шаблоны

`mgt-people`Поддерживает несколько [шаблонов](../templates.md) , которые можно использовать для замены определенных частей компонента. Чтобы указать шаблон, включите элемент в `<template>` компонент и задайте `data-type` для него одно из следующих значений.

| Тип данных | Контекст данных | Описание |
| --- | --- | --- |
| `default` | `people`: список объектов Person | Шаблон по умолчанию заменяет весь компонент своим собственным. |
| `person` | `person`: объект Person | Шаблон, используемый для отображения каждого пользователя. |
| `overflow` | `people`: список объектов Person <br> `max`: количество показанных пользователей <br> `extra`: количество дополнительных людей | Шаблон, используемый для отображения числа за пределами максимального значения справа от списка людей. |
| `no-data` | Контекст данных не передается | Шаблон, используемый, если данные недоступны. |
| `loading` | Контекст данных не передается | Шаблон, используемый при загрузке состояния компонента.

В приведенных ниже примерах показано, как использовать `person` шаблон.

```html
<mgt-people>
  <template>
    <ul><li data-for="person in people">
      <mgt-person person-query="{{ person.userPrincipalName }}"></mgt-person>
      <h3>{{ person.displayName }}</h3>
      <p>{{ person.jobTitle }}</p>
      <p>{{ person.department }}</p>
    </li></ul>
  </template>
</mgt-people>
```

## <a name="microsoft-graph-permissions"></a>Разрешения Microsoft Graph

В этом компоненте используются следующие API и разрешения Microsoft Graph:

| Ресурс | Разрешение |
| - | - |
| [/ме/пеопле](/graph/api/user-list-people?view=graph-rest-1.0) | `People.Read` |

При использовании шаблонов по умолчанию требуются дополнительные API и разрешения. В шаблоне по умолчанию для этого компонента используется компонент " [упр. Person](person.md) ", для которого требуется следующее.

| Ресурс | Разрешение |
| - | - |
| [/Users](/graph/api/user-list?view=graph-rest-1.0) | Users. ReadBasic. ALL |
| [/ме/календарвиев](/graph/api/user-list-contacts?view=graph-rest-1.0) | Contacts.Read |

## <a name="extend-for-more-control"></a>Расширение для дополнительных элементов управления

Для более сложных сценариев или для действительно настраиваемого пользовательского интерфейса этот компонент предоставляет несколько `protected render*` способов переопределения в расширениях компонентов.

| Метод | Описание |
| - | - |
| рендерлоадинг | Отображает состояние загрузки. |
| рендернодата | Отрисовывает пустое состояние данных. |
| рендерпеопле | Отрисовывает список людей вплоть до `show-max` значения. |
| рендерперсон | Отрисовывает отдельного человека. |
| рендероверфлов | Отрисовывает представление оставшихся пользователей за пределами `show-max` значения. |
