<span data-ttu-id="74251-p102">Запросы изменений позволяют приложениям обнаруживать новые, обновленные и удаленные сущности, не считывая целевой ресурс полностью при каждом запросе. ## <a name="use-delta-query-to-track-changes-in-a-resource-collection"></a><span data-ttu-id="74251-107">Отслеживание изменений в коллекции ресурсов с помощью запроса изменений</span><span class="sxs-lookup"><span data-stu-id="74251-107">Use delta query to track changes in a resource collection</span></span>

<span data-ttu-id="74251-108">Ниже представлен типичный шаблон вызова.</span><span class="sxs-lookup"><span data-stu-id="74251-108">The typical call pattern is as follows:</span></span>

1. <span data-ttu-id="74251-109">Для начала приложение выполняет запрос GET с функцией delta для нужного ресурса.</span><span class="sxs-lookup"><span data-stu-id="74251-109">The application begins by calling a GET request with the delta function on the desired resource.</span></span>
2. <span data-ttu-id="74251-110">Microsoft Graph отправит ответ, содержащий нужный ресурс и [маркер состояния](#state-tokens).</span><span class="sxs-lookup"><span data-stu-id="74251-110">Microsoft Graph sends a response containing the requested resource and a [state token](#state-tokens).</span></span>

     <span data-ttu-id="74251-p103">-  Если возвращается URL-адрес `nextLink`, это означает, что во время сеанса могли быть получены не все страницы данных. Для получения всех страниц данных приложение продолжает отправлять запросы, используя URL-адрес `nextLink`, пока в отклик не будет включен URL-адрес `deltaLink`.</span><span class="sxs-lookup"><span data-stu-id="74251-p103">a.  If a `nextLink` URL is returned, there may be additional pages of data to be retrieved in the session. The application continues making requests using the `nextLink` URL to retrieve all pages of data until a `deltaLink` URL is returned in the response.</span></span>

     <span data-ttu-id="74251-p104">- Если возвращается URL-адрес `deltaLink`, это означает, что больше нет данных о текущем состоянии ресурса. 3. <span data-ttu-id="74251-p105">Когда приложению требуется узнать об изменениях ресурса, оно совершает новый запрос, используя URL-адрес `deltaLink`, полученный на шаге 2. 4. <span data-ttu-id="74251-119">Microsoft Graph возвращает описание изменений ресурса с момента последнего запроса вместе с URL-адресом `nextLink` или `deltaLink`.</span><span class="sxs-lookup"><span data-stu-id="74251-119">Microsoft Graph returns a response describing changes to the resource since the previous request, and either a `nextLink` URL or a `deltaLink` URL.</span></span>

><span data-ttu-id="74251-120">**Примечание.** Ресурсы, хранящиеся в Azure Active Directory (например, пользователи и группы), поддерживают сценарии "синхронизировать с этого момента".</span><span class="sxs-lookup"><span data-stu-id="74251-120">**Note:** Resources stored in Azure Active Directory (such as users and groups) support "sync from now" scenarios.</span></span> <span data-ttu-id="74251-121">Это позволяет пропустить шаги 1 и 2, описанные выше (если вы не заинтересованы в получении полного состояния ресурса) и вместо этого запросить самую новую ссылку `deltaLink`.</span><span class="sxs-lookup"><span data-stu-id="74251-121">This allows you to skip steps 1 and 2 above (if you are not interested in retrieving the full state of the resource) and ask for the latest `deltaLink` instead.</span></span> <span data-ttu-id="74251-122">Добавьте `$deltaToken=latest` к функции `delta`, и ответ будет содержать ссылку `deltaLink`, но не будет содержать данные ресурсов.</span><span class="sxs-lookup"><span data-stu-id="74251-122">Append `$deltaToken=latest` to the `delta` function and the response will contain a `deltaLink` and no resource data.</span></span>

### <a name="state-tokens"></a><span data-ttu-id="74251-123">Маркеры состояния</span><span class="sxs-lookup"><span data-stu-id="74251-123">State tokens</span></span>

<span data-ttu-id="74251-p107">Ответ GET на запрос изменений всегда включает URL-адрес, указанный в заголовке `nextLink` или `deltaLink`. URL-адрес `nextLink` включает маркер _skipToken_, а URL-адрес `deltaLink` — _deltaToken_.</span><span class="sxs-lookup"><span data-stu-id="74251-p107">A delta query GET response always includes a URL specified in a `nextLink` or `deltaLink` response header. The `nextLink` URL includes a _skipToken_, and a `deltaLink` URL includes a _deltaToken_.</span></span>

<span data-ttu-id="74251-p108">Эти маркеры непрозрачны для клиента. Вот что вам нужно знать о них:</span><span class="sxs-lookup"><span data-stu-id="74251-p108">These tokens are opaque to the client. The following details are what you need to know about them:</span></span>

- <span data-ttu-id="74251-128">Каждый маркер отражает состояние и представляет моментальный снимок ресурса в этом цикле отслеживания изменений.</span><span class="sxs-lookup"><span data-stu-id="74251-128">Each token reflects the state and represents a snapshot of the resource in that round of change tracking.</span></span>

- <span data-ttu-id="74251-p109">Маркеры состояния также кодируют и включают другие параметры запроса (такие как `$select`), указанные в исходном запросе изменений. Таким образом, не обязательно повторять их в последующих запросах изменений.</span><span class="sxs-lookup"><span data-stu-id="74251-p109">The state tokens also encode and include other query parameters (such as `$select`) specified in the initial delta query request. - <span data-ttu-id="74251-131">Совершая запрос изменений, вы можете скопировать и применить URL-адрес `nextLink` или `deltaLink` при следующем вызове функции **delta**, не проверяя содержимое URL-адреса, в том числе маркер состояния.</span><span class="sxs-lookup"><span data-stu-id="74251-131">When carrying out delta query, you can copy and apply the `nextLink` or `deltaLink` URL to the next **delta** function call without having to inspect the contents of the URL, including its state token.</span></span>

### <a name="optional-query-parameters"></a><span data-ttu-id="74251-132">Необязательные параметры запросов</span><span class="sxs-lookup"><span data-stu-id="74251-132">Optional query parameters</span></span>

<span data-ttu-id="74251-p110">Если клиент использует параметр запроса, он должен быть указан в исходном запросе. <span data-ttu-id="74251-137">Для пользователей и групп действуют ограничения на применение некоторых параметров запросов:</span><span class="sxs-lookup"><span data-stu-id="74251-137">For users and groups, there are restrictions on using some query parameters:</span></span>

- <span data-ttu-id="74251-p111">Если используется параметр запроса `$select`, это означает, что клиент предпочитает отслеживать изменения только для тех свойств или связей, которые указаны в операторе `$select`. - <span data-ttu-id="74251-140">`$expand` поддерживается только для навигационных свойств `manager` и `members` для пользователей и групп соответственно.</span><span class="sxs-lookup"><span data-stu-id="74251-140">`$expand` is only supported for the `manager` and `members` navigational property for users and groups respectively.</span></span>
