# <a name="partnering-with-the-microsoft-graph-security-api--technology-partner-opportunities"></a>Партнерские организации с помощью API безопасности Microsoft Graph — возможности для партнеров по технологиям

В этой статье описываются возможности, активизируемые с помощью API безопасности Microsoft Graph и предназначенные для помощи руководителям и бизнес-задачам, которые помогут руководителям и бизнес-разработкам понять пути инвестиций и предоставить представление о партнерах по ценности.

## <a name="background"></a>Фон

Большинство организаций работают с большими объемами данных безопасности и содержат десятки решений по обеспечению безопасности на предприятии, что делает задачу интеграции различных продуктов и служб в сложное и сложное. Эти трудности мешают быстрому переходу между организациями при обнаружении и устранение угроз в мире с быстрым перемещением, нарушениями атак.

Технологические партнеры могут интегрироваться с платформой Майкрософт с помощью API безопасности Microsoft Graph, чтобы устранить эти проблемы с клиентами.

## <a name="introduction-to-the-microsoft-graph-security-api"></a>Общие сведения об интерфейсе API безопасности Microsoft Graph

API безопасности Microsoft Graph — это единый API, обеспечивающий стандартный интерфейс и универсальную схему для интеграции оповещений системы безопасности и логики безопасности из нескольких источников, обогащения оповещений и данных с контекстными сведениями, а также для автоматизации безопасности функционировани.

API безопасности является частью Microsoft Graph, который является единым API REST для интеграции данных и аналитики от Майкрософт и партнерских продуктов и служб. С помощью Microsoft Graph клиенты и партнеры могут быстро создавать решения, которые однократно проходят проверку подлинности и использовать один вызов API для доступа к ценным сведениям по безопасности из нескольких решений для обеспечения безопасности. Дополнительное значение обнаруживается при изучении других [сущностей Microsoft Graph](https://docs.microsoft.com/graph/overview) (Office 365, Azure Active Directory, Intune и т. д.) для связывания бизнес-контекста с ценными сведениями по безопасности.

Корпорация Майкрософт включает интеграцию с технологическими партнерами двумя основными способами.

1. Как потребитель информации из Microsoft Graph, вы можете расширить свои решения с помощью данных, включенных в Microsoft Graph, а также с помощью API Microsoft Graph, чтобы выполнять задачи от имени клиента.
2. Вы также можете добавлять свои оповещения и действия в Microsoft Graph наряду с поставщиками корпорации Майкрософт.

## <a name="featured-technology-partner-scenarios"></a>Рекомендуемые сценарии для технологических партнеров

Ниже приведены три основных преимущества, которые можно создать с помощью интеграции с API безопасности Microsoft Graph:

1. Ваши клиенты получают преимущества от улучшения эффективности и работы по обеспечению безопасности.
2. Ваши клиенты получают преимущества от обширных сведений, предоставляемых вашими и другими интегрированными партнерами.
3. Проектирование технологических партнеров для технологических партнеров упрощено, а значение клиента увеличено с помощью интеграции с API безопасности Microsoft Graph.

### <a name="enhance-threat-protection-with-the-microsoft-graph-security-api"></a>Улучшение защиты от угроз с помощью API безопасности Microsoft Graph

*Предоставление более простой интеграции оповещений системы безопасности для информирования об обнаружении и отклике угроз.*

- Сопоставьте оповещения и обнаружения от поставщиков безопасности Microsoft Graph с определениями, чтобы упростить результаты расследования и обеспечить поддержку автоматизации.
- Обнаружение и контекст доступа с помощью Microsoft Graph для оптимизации отклика от угроз — рассмотрения, исследования, исправления.
- Доступ к ресурсам (хэш, IP, URL-адрес, домен и т. д.) для защиты от вредоносных действий пользователя.

### <a name="streamline-it-and-security-management"></a>Оптимизация ИТ и управления безопасностью

*Обеспечение большей прозрачности и упрощения управления жизненным циклом инцидентов.*

- Объединение оповещений от нескольких поставщиков для создания инцидентов.
- Доступ к дополнительному контексту для информирования об приоритизации и ответе.
- Синхронизация состояния оповещений по системам, управляющим оповещениями.
- Получите представление о безопасности и рекомендации по их улучшению с учетом безопасной оценки.

### <a name="share-threat-intelligence-to-enable-custom-detections"></a>Совместное использование логики операций с угрозами для включения пользовательских обнаружений

*Используйте логику операций с угрозами для обнаружения в решениях Майкрософт пользовательских обнаружений.*

- Автоматически отправлять индикаторы угроз в решения безопасности Майкрософт, чтобы включить оповещения, блокировать или разрешать действия.
- Включите действие SWIFT, чтобы защититься от новых угроз, таких как блочный файл, URL-адрес, домен, IP-адрес в средствах безопасности и рабочих процессах.
- Пользовательская ЛОГИКа операций, предоставленных клиентом, используется только для поставки клиента, а не для других клиентов корпорации Майкрософт.

## <a name="technical-integrations-overview"></a>Обзор технического взаимодействия

Возможности партнерства по интерфейсу API безопасности Microsoft Graph становятся доступными через два основных пути интеграции, которые можно использовать независимо или вместе.  Мы будем структурировать требования высокого уровня и предоставили сведения о том, как подумать о наличии дел в этих путях здесь, но подробные технические объяснения остаются в документах, указанных далее в этом документе.

Поддерживаемые сущности:

- **Оповещения** — это "выводы с влиянием безопасности", а не необработанные данные журнала или другие несвязанные данные. [Подробнее](https://aka.ms/graphsecurityalerts).
- **Индикаторы угроз**, также называемые индикаторами компромиссов или иокс, представляют собой данные об известных угрозах, такие как вредоносные файлы, URL-адреса, домены и IP-адреса. Клиенты могут создавать индикаторы с помощью внутренней системы анализа угроз или получать индикаторы из сообществ, лицензированных веб-каналов и других источников. [Подробнее](https://aka.ms/graphsecuritytiindicators).
- **Действия по обеспечению безопасности** позволяют технологическим партнерам предоставлять функциональные возможности через график.  Например, если решение для обеспечения безопасности поддерживает возможность блокировки IP-адресов, вы можете предоставить возможность "Block IP" в графе. Другие продукты API для обеспечения безопасности Graph могут вызывать ваше действие через график. [Подробнее](https://aka.ms/graphsecurityaction).

### <a name="integrate-your-application-with-the-microsoft-graph-security-api"></a>Интеграция приложения с помощью API безопасности Microsoft Graph

Все интегрированные приложения [должны быть зарегистрированы](https://docs.microsoft.com/graph/auth-register-app-v2) в Microsoft Graph. Оба приложения, используемые одним клиентом, а также поддерживаемые многими клиентами (с несколькими клиентами).  В любом случае клиент должен предоставить согласие для вашего приложения. При вызове Microsoft Graph каждый запрос из приложения будет содержать идентификатор приложения и клиента, вызываемого от имени. Поддерживаются следующие типы запросов:

- **Получение оповещений** — получение данных оповещения с фильтрацией по мере необходимости.  Например: Показать все оповещения с высоким приоритетом или "все оповещения с высоким приоритетом" для определенного пользователя, ведущего приложения и т. д.
- **Обновление состояния оповещения** — включение управления жизненным циклом оповещений.  Например: Установка состояния оповещения "разрешено" из "выполняется" или добавление комментариев к оповещению.
- **Получение показателя безопасности** — Оценка Microsoft Secure — это значение типа "кредитоспособность" для конфигураций безопасности продуктов Майкрософт.
- **Подписываться** на уведомления об изменениях оповещений или запросов.
- **Пользовательские индикаторы угроз для веб-каналов** — автоматически отправлять индикаторы угроз в решения безопасности Майкрософт, чтобы включить оповещения, блокировать или разрешать действия. Использование API безопасности Microsoft Graph напрямую или использование интеграции с ведущими платформами системы анализа угроз.
- **Вызвать действие по обеспечению безопасности Microsoft Graph** — немедленно предпринимать меры для защиты от угроз с помощью объекта безопасности Microsoft Graph [секуритяктионс](https://aka.ms/graphsecurityaction) .

### <a name="enable-others-to-integrate-with-your-products-through-the-microsoft-graph-security-api"></a>Разрешить другим пользователям интеграцию с продуктами с помощью API безопасности Microsoft Graph

Поставщики безопасности Microsoft Graph предоставляют другим пользователям оповещения о безопасности, доступные через Microsoft Graph.  У продуктов Майкрософт, которые создают оповещения системы безопасности, есть поставщики, которые открывают соответствующие оповещения Microsoft Graph. Кроме того, API безопасности Microsoft Graph позволяет внешним поставщикам, что позволяет вам, как партнер Майкрософт по технологиям, предоставлять доступ к соответствующим оповещениям системы безопасности из ваших приложений в Microsoft Graph для использования пользователями. Кроме оповещений, Microsoft Graph Security Секуритяктионс позволяет партнерским партнерам предоставлять функциональные возможности с помощью Microsoft Graph.  Например, если решение для обеспечения безопасности поддерживает возможность блокировки IP-адресов, вы можете предоставить "Block IP" как возможность в Microsoft Graph. Другие продукты безопасности Microsoft Graph могут вызывать ваше действие через Microsoft Graph.

Поставщик безопасности Microsoft Graph по сути является облачной конечной точкой, которая реагирует на запросы из API безопасности Microsoft Graph и возвращает соответствующие оповещения системы безопасности или выполняет действия для клиентов. Проверка подлинности клиентов и служб обеспечивает безопасность доступа к оповещениям клиентов и действиям.

Сценарии поставщиков широко отличаются. Проверенный процесс входящей миграции начинается с определения соответствующих сценариев. После согласования сценариев, документации, примеров кода и сред разработки можно поддерживать разработку поставщика безопасности Microsoft Graph.
