# <a name="microsoft-teams-api-overview-preview"></a>Обзор API Microsoft Teams (предварительная версия)

Microsoft Teams — это единый центр для работы в группах и интеллектуального обмена данными. В основе Microsoft Teams лежат мощь и масштаб Office 365 с более чем 120 млн пользователей. В нем имеются средства для совместной работы на основе чатов, собраний, звонков и корпоративные голосовые функции.

## <a name="why-integrate-with-microsoft-teams"></a>Зачем выполнять интеграцию с Microsoft Teams?

Интеграция с Microsoft Teams упрощает создание собственных служб и приложений, позволяет связаться с миллионами корпоративных пользователей и помогает людям совместно выполнять работу. С помощью Microsoft Graph вы можете создавать группы, каналы, сообщения и многое другое, а также управлять ими.

### <a name="use-microsoft-graph-in-any-kind-of-app"></a>Использование Microsoft Graph в приложениях любого типа

Приложения Microsoft Teams, созданные на базе Microsoft Graph, предоставляют рабочим группам новое средство для повышения эффективности совместной работы. С помощью Microsoft Teams пользователи в рабочих группах могут делиться ресурсами, взаимодействовать друг с другом в чате и планировать мероприятия в календаре группы. Повысьте удобство работы с Microsoft Teams, добавив средства автоматизации групп, каналов и бесед с помощью приложений, основанных на API Microsoft Teams.

Веб-сайты, службы и собственные приложения платформы не выполняются в пользовательском интерфейсе Microsoft Teams, но их можно использовать для вызова API Microsoft Teams, которые позволяют использовать сценарии автоматизации Microsoft Teams.

**Типы приложений, которые разрешено использовать в Microsoft Teams**

![Вызов API Microsoft Teams из вкладок, ботов, веб-сайтов и служб](images/TeamsAppEndpoints.png)

Эти средства совместной работы включают вкладки или приложения-боты с поддержкой Microsoft Graph, работающие в Microsoft Teams. Кроме того, вы можете вызывать Microsoft Graph из-за пределов приложения Microsoft Teams, например с веб-сайта или из веб-службы. Если вы уже включили поддержку Microsoft Graph на своем веб-сайте, вы можете использовать ее для работы с Microsoft Teams. Для этого с помощью [платформы разработки Microsoft Teams](https://docs.microsoft.com/en-us/microsoftteams/platform/#pivot=home&panel=home-all) создайте приложение вкладки, использующее существующий код веб-сайта.

Если вкладка или приложение-бот Microsoft Teams не подходят для вашего сценария, выберите приложение одного из указанных ниже типов.

|Тип приложения|Описание сценария|
|:-------|:-------------------|
|Вкладки|Показ расширенного контента в Microsoft Teams.|
|Соединители|Публикация расширенных обновлений в каналах.|
|Сообщения с действиями|Добавление расширенного взаимодействия в карточки соединителей.|
|Веб-сайты|Предоставление доступа к расширенному контенту на ваших веб-страницах.|
|Боты|Помощь пользователям в выполнении их задач в беседах.|
|Веб-канал активности|Привлечение пользователей с помощью уведомлений в веб-каналах.|
|Расширения для системы обмена сообщениями|Разрешите пользователям запрашивать расширенные карточки и делиться ими в беседах.|
|Службы|Дополнение клиентских приложений данными из Microsoft Graph через веб-службу.|

### <a name="create-multiple-teams-and-channels"></a>Создание нескольких групп и каналов

Разрешите своим клиентам создавать [группы](../api-reference/beta/resources/team.md) и [каналы](../api-reference/beta/resources/channel.md), связанные с вашим приложением. Упростите создание большого количества групп и наполнение их пользователями и каналами с помощью API Microsoft Teams.

### <a name="automate-team-lifecycles"></a>Автоматизация жизненных циклов групп

Если у вас возникла новая бизнес-проблема, с помощью Microsoft Graph вы можете создать виртуальную группу, [наполнить ее](../api-reference/v1.0/api/group_post_members.md) необходимыми пользователями и настроить каналы для группы. Чтобы начать обсуждение бизнес-проблемы в канале группы, вы можете разместить новую беседу в канале и опубликовать в ней приветственное сообщение для новых участников группы. Если вы хотите привлечь участников группы к обсуждению бизнес-проблемы, добавьте новое мероприятие в календарь группы и пригласите на него участников группы.

После того как бизнес-проблема решена, и вам больше не нужна виртуальная группа, вы можете уничтожить группу с помощью API Microsoft Teams. Если при создании виртуальной группы вы заранее знаете максимальный срок ее работы, настройте для группы [политику завершения срока действия групп в Office 365](https://support.office.com/en-us/article/office-365-group-expiration-policy-8d253fe5-0e09-4b3c-8b5e-f48def064733?ui=en-US&rs=en-US&ad=US), благодаря которой группа будет автоматически удалена согласно параметрам политики.
