## <a name="whats-the-difference-between-oauth-20-and-openid-connect"></a><span data-ttu-id="1d82e-150">В чем разница между OAuth 2.0 и OpenID Connect?</span><span class="sxs-lookup"><span data-stu-id="1d82e-150">What's the difference between OAuth 2.0 and OpenID Connect?</span></span>

<span data-ttu-id="1d82e-p114">OAuth 2.0 — это протокол авторизации. Он определяет, как приложение может получать маркеры доступа, проходя проверку подлинности непосредственно в Azure AD или перенаправляя пользователя, чтобы он прошел проверку подлинности в Azure AD и согласился предоставить разрешения, запрашиваемые приложением. В первом случае приложение получает маркер доступа, с помощью которого оно может вызывать Microsoft Graph от своего имени. Во втором случае приложение получает маркер доступа, с помощью которого оно может вызывать Microsoft Graph от имени пользователя. Однако при использовании протокола OAuth 2.0 приложение не получает никаких сведений о пользователе и о том, как он прошел проверку подлинности Azure AD. Потоки OAuth 2.0 чаще всего используются мобильными и нативными приложениями, которым уже известны личные данные пользователя, либо такими приложениями, как фоновые службы и управляющие программы, которые вызывают Microsoft Graph от своего имени, а не от имени пользователя.</span><span class="sxs-lookup"><span data-stu-id="1d82e-p114">OAuth 2.0 is an authorization protocol. It defines how your app can get access tokens by authenticating directly with Azure AD or by redirecting a user to authenticate with Azure AD and consent to the permissions your app requests. In the first case, your app gets an access token that it can use to call Microsoft Graph as itself. In second case, your app gets an access token that it can use to call Microsoft Graph on behalf of a user. With OAuth 2.0, however, your app does not receive any information about the user or how they were authenticated by Azure AD. OAuth 2.0 flows are most often used by mobile or native apps, which already know the identity of the user; or by apps like background services or daemons, which call Microsoft Graph under their own identity and not on behalf of a user.</span></span>

<span data-ttu-id="1d82e-p115">OpenID Connect расширяет протокол OAuth 2.0, добавляя уровень удостоверений. При использовании OpenID Connect, помимо маркера доступа, приложение также может получить маркер идентификатора из службы Azure AD. Маркеры идентификаторов OpenID Connect содержат утверждения об удостоверении пользователя и сведения о том, как и где он прошел проверку подлинности. Потоки OpenID Connect обычно используются веб-приложениями, в том числе одностраничными (SPA). С помощью маркера идентификатора эти приложения могут настраивать свое поведение для пользователя, для которого запрашивался маркер доступа. Кроме того, во многих случаях вход пользователей поручается службе Azure AD. При этом становятся доступны такие возможности, как единый вход.</span><span class="sxs-lookup"><span data-stu-id="1d82e-p115">OpenID Connect extends OAuth 2.0 to provide an identity layer. With OpenID Connect, in addition to an access token, your app can also get an id token from Azure AD. OpenID Connect id tokens contain claims about the user's identity and details about how and where they were authenticated. OpenID Connect flows are typically used by web apps, including single page apps (SPAs). These apps can use the id token to customize their behavior for the user they've requested an access token for, and, in many cases, will outsource sign-in of their users to Azure AD and enable experiences like Single Sign-on (SSO).</span></span>
