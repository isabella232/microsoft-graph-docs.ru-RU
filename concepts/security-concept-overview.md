# <a name="overview-of-security-in-microsoft-graph-preview"></a>Обзор средств безопасности в Microsoft Graph (ознакомительная версия) 

С помощью Microsoft Graph вы можете подключаться к Intelligent Security Graph, чтобы пользоваться ресурсами корпорации Майкрософт и ее партнеров по обеспечению безопасности и создавать более эффективные решения для реагирования на угрозы. Microsoft Graph также предоставляет доступ к сведениям о рисках для пользователей и учетных записей, обнаруженных службой защиты идентификации в Azure Active Directory (Azure AD), что позволяет интегрировать данные о рисках для учетных записей в приложения для обеспечения безопасности.

## <a name="why-use-the-security-api-and-connect-with-microsoft-intelligent-security-graph"></a>Зачем использовать API безопасности и подключаться к Microsoft Intelligent Security Graph?

Intelligent Security Graph — это единая платформа для борьбы с киберугрозами. Она обеспечивает защиту от угроз в реальном времени для продуктов и услуг Майкрософт, а также поддерживает экосистему интегрированных решений.

[API безопасности в Microsoft Graph](https://aka.ms/graphsecuritydocs) упрощает подключение к этим решениям в Intelligent Security Graph. Он помогает быстрее реализовать и расширить потенциал этих решений.

### <a name="unify-and-standardize-alert-management"></a>Объединение и стандартизация управления оповещениями

Легко сопоставляйте оповещения из разных решений для обеспечения безопасности с помощью общей схемы оповещений. Достаточно один раз написать код интеграции оповещений из любого решения для обеспечения безопасности, интегрированного с Microsoft Graph, и состояние оповещений и назначения будут синхронизированы между всеми решениями. Вы также можете выполнять потоковую передачу оповещений в решения для управления информационной безопасностью и событиями безопасности (SIEM), такие как Splunk и IBM QRadar, через [Azure Monitor](https://docs.microsoft.com/ru-RU/azure/monitoring-and-diagnostics/monitor-stream-monitoring-data-event-hubs#what-can-i-do-with-the-monitoring-data-being-sent-to-my-event-hub).

### <a name="federated-security-aggregation-service"></a>Служба агрегирования федеративных данных о безопасности

Используя API безопасности в качестве службы агрегирования федеративных данных о безопасности, вы можете отправлять запросы всем подключенным поставщикам безопасности и получать агрегированные отклики.

### <a name="unlock-security-context-to-drive-investigation"></a>Разблокировка контекста безопасности для расследований

Подробно изучите ресурсы, связанные с безопасностью (например, пользователей, узлы и приложения), а затем добавьте контекст организации от других поставщиков Microsoft Graph (Azure AD, Microsoft Intune, Office 365), чтобы объединить контексты бизнеса и безопасности для более эффективного реагирования на угрозы.

## <a name="why-use-azure-ad-to-protect-identities-in-your-organization"></a>Зачем использовать Azure AD для защиты удостоверений в организации?

Большинство нарушений безопасности возникают в результате похищения удостоверения пользователя злоумышленником. Взломщики научились поразительно эффективно пользоваться уязвимостями сторонних решений, проводить атаки путем распыления пароля и сложные фишинговые атаки. Это означает, что вам нужно защищать все учетные записи пользователей от таких атак и предпринимать меры, препятствующие несанкционированному использованию скомпрометированных удостоверений.

Azure AD использует адаптивные алгоритмы машинного обучения и эвристический анализ, чтобы определять аномалии, указывающие на возможную компрометацию учетных записей. Используя эти данные, служба защиты идентификации защищает пользователей при помощи политик условного доступа на основе оценки рисков, а также создает отчеты и оповещения об обнаруженных угрозах.

В настоящее время Microsoft Graph предоставляет пользователям Azure AD Premium P1 и P2 возможность запрашивать сведения о рисках, обнаруженных службой защиты идентификации, и использовать эти события в системах SIEM и приложениях для обеспечения безопасности.
