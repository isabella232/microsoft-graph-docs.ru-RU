# <a name="policy-resource-type"></a>Тип ресурса политики

> **Важно!** API бета-версии (/beta) в Microsoft Graph проходят тестирование и могут быть изменены. Использование этих API в производственных приложениях не поддерживается.

Представляет политику Azure AD. Политики, настраиваемые правила, которые могут применяться в приложениях, субъектов-служб, групп или во всей организации, которые они назначены. В настоящее время только для одного типа политики доступна:

- Маркер срока действия-задает продолжительность времени жизни маркеры, выданные для приложений и субъектов-служб.

Эта политика является подробно описаны ниже.

## <a name="token-lifetime-policy"></a>Политика срока жизни маркера
Указывает время существования маркеры, выданные для различных целей. Этот тип политики может быть [назначен](../api/policy-assign.md) для приложения и субъектов-служб. Существует четыре типа маркеры, время существования которых можно настроить. Пары маркер доступа и обновления можно получить во время проверки подлинности с помощью клиента, тогда как пары маркеров идентификатор/сеанса можно получить в ходе проверки подлинности через браузер.

- **Маркер доступа** содержит сведения об удостоверении и права доступа, связанные с учетной записью пользователя, который используется клиентами для доступа к защищенным ресурсам как приложения.
- **Обновление маркеров** получается вместе с маркером доступа когда пользователь подлинность Azure AD через клиент мог получить доступ к защищенному ресурсу. При не отозван или исключаются неиспользуемых для более чем MaxInactiveTime (ниже), его можно использовать для получения новую пару маркер доступа и обновления после истечения срока действия текущего маркера доступа.
- **Идентификатор маркера** ведет себя как маркер доступа, но получить с помощью браузера.
- **Сеанс маркеров** действует как маркер обновления, но полученные в браузере.

## <a name="properties"></a>Свойства
Свойства ниже форм объект JSON, который представляет политики срока жизни маркера. Этот объект JSON должен быть **преобразован в строку с помощью предложения escape-последовательность** для вставки в общих свойств политики «определение». Ниже показан пример.

>Примечание: Все интервалам времени в эти свойства задаются в формате «ДД.чч:мм:сс».

>Примечание: Максимального значения для свойства идентификаторами в «дней» являются одной секунды немного не обозначаемого число дней. Например, указано значение 1 дней «23: 59:59».

| Свойство     | Тип   |Описание| Минимальное значение | Максимальное значение | Значение по умолчанию|
|:---------------|:--------|:----------|:--------|:--------|:----|
|AccessTokenLifetime|Строка|Определяет, как долго **доступа и маркеров безопасности** считается допустимым.|10 минут|1 день|1 час|
|MaxInactiveTime|Строка|Определяет, как старый маркер обновления могут быть перед клиент больше не может использовать ее для извлечения новую пару маркер доступа и обновление для доступа к ресурсу.|10 минут|90 дней|14 дней|
|MaxAgeSingleFactor|Строка|Сколько времени элементы управления пользователя могут продолжить использовать токены обновления для получения нового клиента и обновление маркеров пары после последнего они проверку подлинности успешно только одного раза. Так как проверкой считается менее безопасным, чем многофакторной проверки подлинности, рекомендуется, что эта политика установлена равной значение равно или меньше, чем MultiFactorRefreshTokenMaxAge.|10 минут|пока не отозваны|365 дней или пока не отозваны|
|MaxAgeMultiFactor|Строка|Сколько времени элементы управления пользователя могут продолжить использовать токены обновления для получения нового клиента и обновление маркеров пары после последнего они проверку подлинности успешно с несколькими факторов.|10 минут|пока не отозваны|365 дней или пока не отозваны|
|MaxAgeSessionSingleFactor|Строка|Сколько времени элементы управления пользователя можно продолжить используйте маркеры сеанса, чтобы получить новый идентификатор/сеанса маркеры после последнего выполнения их проверки подлинности успешно только одного раза. Так как проверкой считается менее безопасным, чем многофакторной проверки подлинности, рекомендуется, что эта политика установлена равной значение равно или меньше, чем MultiFactorSessionTokenMaxAge|10 минут|пока не отозваны|365 или пока не отозваны|
|MaxAgeSessionMultiFactor|Строка|Сколько времени элементы управления пользователя можно продолжить используйте маркеры сеанса, чтобы получить новый идентификатор/сеанса маркеры после последнего их подлинность с несколькими факторов.|10 минут|пока не отозваны|365 или пока не отозваны|
|Version|Целое число|Задайте значение 1. Обязательный.|Нет|Нет|Нет|

```json
{
  "definition":["{\"TokenLifetimePolicy\":{\"Version\":1,\"AccessTokenLifetime\":\"8:00:00\",\"MaxInactiveTime\":\"20:00:00\",}}"],
  "displayName":"Test Policy",
  "isOrganizationDefault":false,
  "type":"TokenLifetimePolicy",
}
```
