[!INCLUDE [beta-disclaimer](../../includes/beta-disclaimer.md)]

<span data-ttu-id="273a0-104">Ресурс **call** создается при наличии входящего вызова приложения или при создании приложением нового исходящего вызова с помощью запроса `POST` в `app/calls`.</span><span class="sxs-lookup"><span data-stu-id="273a0-104">The **call** resource is created when there is an incoming call for the application or the application creates a new outgoing call via a `POST` on `app/calls`.</span></span>

<span data-ttu-id="273a0-105">Вызовы можно настраивать как одноранговые или многопользовательские вызовы.</span><span class="sxs-lookup"><span data-stu-id="273a0-105">Calls can be set up as a peer-to-peer or as a multiparty call.</span></span> <span data-ttu-id="273a0-106">Чтобы создать или присоединиться к многопользовательскому вызову, укажите `chatInfo` и `meetingInfo`.</span><span class="sxs-lookup"><span data-stu-id="273a0-106">For creating or joining a multiparty call, supply the `chatInfo` and `meetingInfo`.</span></span> <span data-ttu-id="273a0-107">Если они не указаны, автоматически создается незапланированное собрание.</span><span class="sxs-lookup"><span data-stu-id="273a0-107">If these are not supplied, a new ad hoc meeting is created automatically.</span></span> <span data-ttu-id="273a0-108">Для входящих вызовов записывайте эти значения в высокодоступном хранилище, чтобы приложение повторно присоединилось к вызову в случае своего сбоя.</span><span class="sxs-lookup"><span data-stu-id="273a0-108">For an incoming call, record these values in a highly available store, so that your application to rejoin the call in the event your application crashes.</span></span>
