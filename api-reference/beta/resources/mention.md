# <a name="mention-resource-type"></a>Укажите тип ресурса

> **Важно!** API бета-версии (/beta) в Microsoft Graph проходят тестирование и могут быть изменены. Использование этих API в производственных приложениях не поддерживается.

Представляет уведомления для пользователя, на основе адреса электронной почты контакта. Этот тип уведомления также называется @ упоминания.

Ресурс [сообщений](../resources/message.md) поддерживает **упомянуть**. Он содержит свойство **mentionsPreview** , которое указывает, описанного ли пользователь выполнил вход в этот экземпляр сообщения. Он также включает свойство навигации **упоминания** , которое поддерживает получение сведений о Упоминание или удаление упоминаются в этот экземпляр.

При создании сообщения, приложение можно создать упоминаются в том же `POST` запрос, включая упоминаются в свойстве **упоминания** . С помощью `GET` запрос с `$filter` параметр запроса, приложение может вернуть все сообщения в почтовом ящике пользователь выполнил вход, упомянуть пользователя. A `GET` запрос с `$expand` параметр позволяет приложение разверните все упоминания в определенное сообщение запроса.

Этот механизм, позволяющий приложения задание и извлечение упоминания в сообщениях на включает lightweight уведомления, где пользователя, выполняющего упоминание могут оставаться в существующий контекст (например, при составлении сообщения) при приложение задает свойство базового **упоминания** . Упомянутые лиц можно легко выяснить, если и где упомянутые через `GET` запрашивает с `$filter` или `$expand` параметр запроса.  

Например, в почтовом клиенте Outlook, когда пользователь вводит `@` при записи сообщения Outlook позволяет пользователей выберите или введите имя для выполнения упоминание @. Outlook устанавливает для свойства **упоминания** перед создает и отправляет сообщение или события. Outlook также использует `GET` операции с `$filter` и `$expand` сообщите пользователь выполнил вход искать сообщения, содержащие упоминание пользователя, элемент интерфейса, предупреждающий пользователя элементы действия или обсуждения, что обеспечивает более быстрый отклик.

## <a name="properties"></a>Свойства
| Свойство     | Тип   |Описание|
|:---------------|:--------|:----------|
|application | String | Имя приложения, где создается упоминание. Необязательный атрибут. Не используется и не по умолчанию установлено значение NULL для **сообщения**. |
|clientReference | String | Уникальный идентификатор, представляющий родительский объект экземпляра ресурса. Необязательный атрибут. Не используется и не по умолчанию установлено значение NULL для **сообщения**. |
|createdBy  | [emailAddress](../resources/emailaddress.md) | Сведения о электронной почты пользователя, выполнившего упоминание. |
|createdDateTime  |DateTimeOffset |Дата и время создания упоминаются в клиенте. |
|deepLink | String | Глубокое web ссылка на контексте упоминаются в экземпляре ресурсов. Необязательный атрибут. Не используется и не по умолчанию установлено значение NULL для **сообщения**. |
|id | String| Уникальный идентификатор упоминаются в экземпляре ресурсов.|
|упомянутые | [emailAddress](../resources/emailaddress.md) | Сведения о электронной почты человека, упомянутых выше. Обязательный атрибут. |
|mentionText | String | Необязательный параметр. Не используется и не по умолчанию установлено значение NULL для **сообщения**. Чтобы получить упоминания в сообщении, см **bodyPreview** сообщения вместо этого. |
|serverCreatedDateTime | DateTimeOffset | Дата и время создания упоминание на сервере. Необязательный атрибут. Не используется и не по умолчанию установлено значение NULL для **сообщения**. |
