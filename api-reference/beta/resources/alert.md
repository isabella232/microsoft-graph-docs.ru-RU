# <a name="alert-resource-type"></a>Тип ресурса Alert

[!INCLUDE [beta-disclaimer](../../includes/beta-disclaimer.md)]

Представляет потенциальные проблемы безопасности в клиенте клиента, которые идентифицированы с помощью решений корпорации Майкрософт или партнерской системы безопасности. Используйте оповещения для объединения и оптимизации управления проблемами безопасности во всех интегрированных решениях. Дополнительные сведения см. в примерах запросов в [проводнике Graph](https://developer.microsoft.com/graph/graph-explorer).

Оповещения можно получить у разных поставщиков безопасности, перечисленных в [статье Обзор безопасности Microsoft Graph](security-api-overview.md).

| Метод   | Возвращаемый тип|Описание|
|:---------------|:--------|:----------|
|[получение оповещения](../api/alert-get.md); | [alert](alert.md) |Чтение свойств и связей объекта Alert.|
|[Обновление оповещения](../api/alert-update.md) | [alert](alert.md) |Обновление объекта Alert. |
|[Перечисление оповещений](../api/alert-list.md) | [](alert.md) коллекция Alerts |Получение коллекции объектов Alert.|
|[Обновление оповещений](../api/alert-updatealerts.md)|[](alert.md) коллекция Alerts|Обновление нескольких объектов Alert.|

## <a name="properties"></a>Свойства

| Свойство   | Тип|Описание|
|:---------------|:--------|:----------|
|Активитиграупнаме|String|Имя или псевдоним группы действий (взломщик) это предупреждение имеет атрибут ".|
|assignedTo|String|Имя аналитика, которому назначено оповещение для рассмотрения, исследования или исправления (поддерживается [Обновление](../api/alert-update.md)).|
|Азуресубскриптионид|String|Идентификатор подписки Azure, присутствует, если это оповещение связано с ресурсом Azure.|
|Азуретенантид |String|Идентификатор клиента Azure Active Directory. Обязательный. |
|category|String|Категория оповещения (например, Кредентиалсефт, шантажистом и т. д.).|
|Клоседдатетиме|DateTimeOffset|Время закрытия оповещения. Тип Timestamp представляет сведения о дате и времени с использованием формата ISO 8601, причем всегда используется время в формате UTC. Например, полночь UTC 1 января 2014: `'2014-01-01T00:00:00Z'` (поддерживает [Обновление](../api/alert-update.md)).|
|Клаудаппстатес|Коллекция [клаудаппсекуритистате](cloudappsecuritystate.md)|Сведения о состоянии, связанные с безопасностью, создаваемые поставщиком о облачном приложении/s, связанном с этим предупреждением.|
|comments|Коллекция String|Комментарии, предоставляемые клиентами, в оповещении (для управления оповещениями клиентов) (поддерживается [Обновление](../api/alert-update.md)).|
|confidence|Int32|Уверенность в логике обнаружения (в процентах от 1-100).|
|createdDateTime |DateTimeOffset|Время создания оповещения поставщиком оповещений. Тип Timestamp представляет сведения о времени и дате с использованием формата ISO 8601 (всегда применяется формат UTC). Например, значение полуночи 1 января 2014 г. в формате UTC выглядит так: `'2014-01-01T00:00:00Z'`. Обязательный.|
|description|String|Описание оповещения.|
|Детектионидс|Коллекция String|Набор оповещений, связанных с этой сущностью Alert (каждое оповещение помещается в SIEM в качестве отдельной записи).|
|eventDateTime |DateTimeOffset|Время, когда произошло событие, которое выдается в качестве триггеров для создания оповещения. Тип Timestamp представляет сведения о времени и дате с использованием формата ISO 8601 (всегда применяется формат UTC). Например, значение полуночи 1 января 2014 г. в формате UTC выглядит так: `'2014-01-01T00:00:00Z'`. Обязательный.|
|замечания|alertFeedback|Обратная связь аналитика с оповещением. Возможные значения: `unknown`, `truePositive`, `falsePositive`, `benignPositive`. (поддерживает [Обновление](../api/alert-update.md))|
|Филестатес|Коллекция [объекта filesecuritystate](filesecuritystate.md)|Сведения о состоянии, связанные с безопасностью, создаваемые поставщиком о файлах, связанных с этим предупреждением.|
|Хистористатес|Коллекция [алерсистористате](alerthistorystate.md)| Коллекция **алерсистористатес** , состоящая из журнала аудита всех обновлений, внесенных в оповещение. |
|Хостстатес|Коллекция [хостсекуритистате](hostsecuritystate.md)|Сведения о состоянии, связанные с безопасностью, создаваемые поставщиком о хост-серверах, связанных с этим предупреждением.|
|id |Строка|Идентификатор GUID/уникальный идентификатор, созданный поставщиком. Только для чтения. Обязательный.|
|lastModifiedDateTime|DateTimeOffset|Время последнего изменения объекта оповещения. Тип Timestamp представляет сведения о времени и дате с использованием формата ISO 8601 (всегда применяется формат UTC). Например, значение полуночи 1 января 2014 г. в формате UTC выглядит так: `'2014-01-01T00:00:00Z'`.|
|Малварестатес|Коллекция [объекта malwarestate](malwarestate.md)|Логика защиты от угроз относится к вредоносным программам, связанным с этим предупреждением.|
|Нетворкконнектионс|Коллекция [networkConnection](networkconnection.md)|Сведения о состоянии, связанные с безопасностью, которые создает поставщик о сетевых подключениях, связанных с этим предупреждением.|
|процесса|Коллекция [Process](process.md)|Сведения о состоянии, связанные с безопасностью, создаваемые поставщиком о процессе или процессах, связанных с этим предупреждением.|
|Рекоммендедактионс|Коллекция String|Рекомендуемые действия поставщика или поставщика, которые необходимо выполнить в результате оповещения (например, изолируйте компьютер, enforce2FA, переобразировать узел).|
|Регистрикэйстатес|Коллекция [регистрикэйстате](registrykeystate.md)|Сведения о состоянии, связанные с безопасностью, создаваемые поставщиком о разделах реестра, связанных с этим предупреждением.|
|серьезность |alertSeverity|Серьезность оповещения — устанавливается поставщиком или поставщиком. Возможные значения: `unknown`, `informational`, `low`, `medium`, `high`. Обязательный.|
|Саурцематериалс|Коллекция String|Гиперссылки (URI) к исходным материалам, связанным с предупреждением, например к пользовательскому интерфейсу поставщика для оповещений или поиска в журнале и т. д.|
|status |alertStatus|Состояние жизненного цикла оповещений (Stage). Возможные значения: `unknown`, `newAlert`, `inProgress`, `resolved`. (поддерживает [Обновление](../api/alert-update.md)). Обязательный.|
|tags|Коллекция String|Определяемые пользователем метки, которые могут быть применены к оповещению и могут использоваться в качестве условий фильтра (например, "Хва", "показано" и т. д.) (поддерживает [Обновление](../api/alert-update.md)).|
|title |String|Заголовок оповещения. Обязательный.|
|триггеров|Коллекция [алерттригжер](alerttrigger.md)|Сведения о свойствах, связанных с безопасностью, для определенных свойств, которые запускают оповещение (свойства, отображаемые в оповещении). Оповещения могут содержать сведения о нескольких пользователях, хостах, файлах, IP-адресах. В этом поле указываются свойства, активируемые при создании оповещения.|
|Усерстатес|Коллекция [усерсекуритистате](usersecuritystate.md)|Сведения о состоянии, связанные с безопасностью, создаваемые поставщиком об учетных записях пользователей, связанных с этим предупреждением.|
|Вендоринформатион |[Секуритивендоринформатион](securityvendorinformation.md)|Сложный тип, содержащий сведения о продукте, поставщике и подобеспечении безопасности (например, Vendor = Microsoft; Provider = защитник Windows ATP; подПредоставление = AppLocker). Обязательный.|
|Вулнерабилитистатес|Коллекция [объекта vulnerabilitystate](vulnerabilitystate.md)|Сведения о угрозах, относящиеся к одной или нескольким уязвимостям, связанным с этим предупреждением.|
