# <a name="list-subscriptions"></a>Перечислить подписки

Выводит свойства и отношения подписок типа веб-перехватчиков на основе идентификатора приложения, пользователя и роли пользователя по отношению к клиенту.

## <a name="permissions"></a>Разрешения

Этот API поддерживает следующие области разрешений. Для получения дополнительных сведений, а также чтобы узнать, как выбирать разрешения, см. [Разрешения](../../../concepts/permissions_reference.md).

| Тип разрешения  | Разрешения (в порядке повышения привилегий)  |
|:---------------- |:-------------------------------------------- |
| [Делегированные](../../../concepts/auth_v2_user.md) (рабочая или учебная учетная запись) | Роли, необходимые для [создания подписки](subscription_get.md) или Subscriptions.Read.All (см. ниже). |
| [Делегированные](../../../concepts/auth_v2_user.md) (личная учетная запись Майкрософт) | Роли, необходимые для [создания подписки](./subscription_get.md) или Subscriptions.Read.All (см. ниже). |
| [Приложение](../../../concepts/auth_v2_service.md) | Роли, необходимые для [создания подписки](./subscription_get.md). |

Результаты ответов определяются по контексту вызывающего приложения. Ниже приведен перечень распространенных сценариев:

### <a name="basic-scenarios"></a>Основные сценарии

Чаще всего приложение пытается показать подписки, которые оно изначально создало для пользователя, вошедшего в систему, или для всех пользователей в каталоге (рабочие/учебные учетные записи). Эти сценарии не требуют каких-либо специальных разрешений, помимо тех, которые приложение изначально использовало для создания своих подписок.

| Контекст вызывающего приложения | Ответ содержит |
|:-----|:---------------- |
| Приложение вызывает от имени выполнившего вход пользователя (делегированное разрешение). <br/>- и -<br/>Приложение изначально имеет разрешение, которое необходимо, чтобы [создать список](subscription_post_subscriptions.md).<br/><br/>Примечание. Это применимо к личным учетным записям Майкрософт и рабочим/учебным учетным записям. | Подписки, созданные **этим приложением** только для выполнившего вход пользователя. |
| Приложение вызывает от своего имени (разрешение приложения).<br/>- и -<br/>Приложение изначально имеет разрешение, которое необходимо, чтобы [создать список](subscription_post_subscriptions.md).<br/><br/>Примечание. Это применимо к только рабочим/учебным учетным записям.| Подписки, созданные **этим приложением** для себя или для какого-либо пользователя в каталоге.|

### <a name="advanced-scenarios"></a>Дополнительные сценарии

В некоторых случаях приложения пытается показать подписки, созданные другими приложениями. Например, пользователю необходимо просмотреть все подписки, созданные каким-либо приложением от его имени. Или администратору может потребоваться просмотреть все подписки от всех приложений в его каталоге.
Для таких сценариев требуется делегированное разрешение Subscription.Read.All.

| Контекст вызывающего приложения | Ответ содержит |
|:-----|:---------------- |
| Приложение вызывает от имени выполнившего вход пользователя (делегированное разрешение). *Пользователь не является администратором*. <br/>- и -<br/>Приложение имеет разрешение Subscription.Read.All<br/><br/>Примечание. Это применимо к личным учетным записям Майкрософт и рабочим/учебным учетным записям. | Подписки, созданные **каким-либо приложением** только для выполнившего вход пользователя. |
| Приложение вызывает от имени выполнившего вход пользователя (делегированное разрешение). В списке приложений выберите пункт *Администратор*.<br/>- и -<br/>Приложение имеет разрешение Subscription.Read.All<br/><br/>Примечание. Это применимо к только рабочим/учебным учетным записям. | Подписки, созданные **каким-либо приложением** для **какого-либо пользователя** в каталоге.|

## <a name="http-request"></a>HTTP-запрос

<!-- { "blockType": "ignored" } -->

```http
GET /subscriptions
```

## <a name="optional-query-parameters"></a>Необязательные параметры запросов

Этот метод не поддерживает [Параметры запроса OData](http://developer.microsoft.com/en-us/graph/docs/overview/query_parameters) для настройки ответа.

## <a name="request-headers"></a>Заголовки запросов

| Имя       | Тип | Описание|
|:-----------|:------|:----------|
| Авторизация  | строка  | Bearer {токен}. Обязательный. |

## <a name="request-body"></a>Текст запроса

Не указывайте тело запроса для этого метода.

## <a name="response"></a>Ответ

При успешном выполнении этот метод возвращает `200 OK` код ответа и список [   объектов](../resources/subscription.md)  в теле ответа.
