# <a name="working-with-azure-active-directory-resources-in-microsoft-graph"></a>Работа с ресурсами Azure Active Directory в Microsoft Graph

Microsoft Graph обеспечивает доступ к ресурсам [Azure Active Directory (Azure AD)](https://docs.microsoft.com/azure/active-directory/active-directory-whatis), которые можно использовать для управления ролями администратора, приглашения внешних пользователей в организацию и, если вы [поставщик облачных решений (CSP)](https://partner.microsoft.com/cloud-solution-provider), управления данными клиента. Microsoft Graph также предоставляет несколько методов, которые могут использоваться многими приложениями, например для поиска информации о транзитивных группах и ролях пользователей. 

> **Примечание.** Некоторые ресурсы Azure AD описаны в других разделах справочника по API ([пользователи](users.md), [группы](group.md)).


## <a name="authorization"></a>Авторизация
 
Для вызова API Microsoft Graph для ресурсов Azure AD приложению необходимы соответствующие разрешения.  Для многих API, предоставляемых для ресурсов Azure AD, требуется одно из [разрешений _Directory_](../../../concepts/permissions_reference.md#directory-permissions).  Разрешения _Directory_ дают широкие права и всегда предоставляются администратором. 

Существуют два типа разрешений: делегированные и для приложения. Если приложение вызывает API от имени пользователя, ему нужны делегированные разрешения. Если оно вызывает API от своего имени, ему нужны разрешения для приложения. Последний сценарий обычно касается серверных служб и управляющих программ. Дополнительные сведения о делегированных разрешениях и разрешениях для приложений см. в справочнике по [разрешениям](../../../concepts/permissions_reference.md). 

Если приложение вызывает API AD Azure от имени пользователя, этому пользователю должна быть назначена соответствующая [роль администратора](https://docs.microsoft.com/azure/active-directory/active-directory-assign-admin-roles).
