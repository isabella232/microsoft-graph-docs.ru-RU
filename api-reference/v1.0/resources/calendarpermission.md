# <a name="calendarpermission-resource-type"></a>Тип ресурса Календарпермиссион

Разрешения пользователя, к которому открыт общий доступ к календарю или который был делегирован в клиенте Outlook.

Получение, обновление и удаление разрешений календаря поддерживается только от имени владельца календаря.

Получение разрешений календаря для календаря от имени объекта Share или делегата возвращает пустой набор разрешений календаря.

После настройки общего доступа или делегирования для календаря можно [Обновить](../api/calendarpermission-update.md) только свойство **Role** , чтобы изменить разрешения для общего доступа или делегата. Невозможно **Обновить** свойства **алловедролес**, **EmailAddress**, **исинсидеорганизатион и**. **isRemovable** Чтобы изменить эти свойства, необходимо [Удалить](../api/calendarpermission-delete.md) соответствующий объект **календарпермиссион** и создать еще одну общую папку или делегат в клиенте Outlook.

## <a name="properties"></a>Свойства

| Свойство     | Тип        | Описание |
|:-------------|:------------|:------------|
|алловедролес|Коллекция [календарролетипе](#calendarroletype-values)| Список разрешенных для общего доступа или делегирования уровней разрешений для календаря. Возможные значения: `none`, `freeBusyRead`, `limitedRead`, `read`, `write`, `delegateWithoutPrivateEventAccess`, `delegateWithPrivateEventAccess`, `custom`.|
|emailAddress|[emailAddress](emailaddress.md)| Представляет общий доступ или представитель, имеющий доступ к календарю. Для общего доступа к общему ресурсу "Моя организация" свойство **Address** имеет значение null. Только для чтения. |
|id|String| Уникальный идентификатор пользователя (общего доступа или представителя), к которому открыт общий доступ к календарю. Только для чтения.|
|исинсидеорганизатион|Логический| Имеет значение true, если пользователь в контексте (общий доступ или представитель) находится в той же организации, что и владелец календаря.|
|isRemovable|Логический| `True`, если пользователь может быть удален из списка общих или делегатов для указанного календаря, `false` в противном случае. Пользователь "Моя организация" определяет разрешения, которые пользователи в организации имеют в указанном календаре. Вы не можете удалить "Моя организация" как общий доступ к календарю.|
|role|[календарролетипе](#calendarroletype-values)| Текущий уровень разрешений для общего доступа к календарю или представителем. |

### <a name="calendarroletype-values"></a>значения Календарролетипе

| Значения        | Описание |
|:--------------|:------------|
| нет | Доступ к календарю для пользователя не предоставляется. |
| фрибусиреад | Пользователь — это общий доступ, который может просматривать сведения о доступности владельца в календаре. |
| лимитедреад | Пользователь предоставляет доступ к сведениям о доступности, а также заголовках и расположениях событий в календаре. |
| прочитан | Пользователь предоставляет общий доступ, который может просматривать все сведения о событиях в календаре, за исключением частных событий владельца. |
| понижен | Пользователь — это общий доступ, который может просматривать все сведения (кроме частных событий) и изменять события в календаре. |
| делегатевисаутпривативентакцесс | Пользователь — это делегат с доступом для записи, но не может просматривать сведения о закрытых событиях владельца в календаре. |
| делегатевиспривативентакцесс | Пользователь — это представитель, имеющий доступ на запись и позволяющий просматривать сведения о частных событиях владельца в календаре. |
| собственный | Пользователь обладает пользовательскими разрешениями на доступ к календарю. |
