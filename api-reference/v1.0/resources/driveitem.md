# <a name="driveitem-resource-type"></a><span data-ttu-id="cdc57-104">Тип ресурса driveItem</span><span class="sxs-lookup"><span data-stu-id="cdc57-104">driveItem resource type</span></span>

<span data-ttu-id="cdc57-105">Пространство имен: microsoft.graph</span><span class="sxs-lookup"><span data-stu-id="cdc57-105">Namespace: microsoft.graph</span></span>

<span data-ttu-id="cdc57-p102">Ресурс **driveItem** представляет файл, папку или другой элемент, хранящийся на диске. <span data-ttu-id="cdc57-108">Обратиться к ресурсу **driveItem** можно двумя основными способами:</span><span class="sxs-lookup"><span data-stu-id="cdc57-108">There are two primary ways of addressing a **driveItem** resource:</span></span>

* <span data-ttu-id="cdc57-109">по уникальному идентификатору **driveItem** с помощью `drive/items/{item-id}`;</span><span class="sxs-lookup"><span data-stu-id="cdc57-109">By the **driveItem** unique identifier using `drive/items/{item-id}`</span></span>

<span data-ttu-id="cdc57-111">У ресурсов **driveItem** есть аспекты, смоделированные как свойства, которые предоставляют данные об идентификаторах и возможностях ресурса driveItem.</span><span class="sxs-lookup"><span data-stu-id="cdc57-111">**driveItem** resources have facets modeled as properties that provide data about the driveItem's identities and capabilities.</span></span>
<span data-ttu-id="cdc57-112">Пример:</span><span class="sxs-lookup"><span data-stu-id="cdc57-112">For example:</span></span>

* <span data-ttu-id="cdc57-113">У папок есть [**аспект folder**][folder].</span><span class="sxs-lookup"><span data-stu-id="cdc57-113">Folders have a [**folder facet**][folder]</span></span>
* <span data-ttu-id="cdc57-114">У файлов есть [**аспект file**][file].</span><span class="sxs-lookup"><span data-stu-id="cdc57-114">Files have a [**file facet**][file].</span></span>

* <span data-ttu-id="cdc57-116">У изображений, полученных с помощью камеры (фотографий), есть [**аспект photo**][photo], который определяет элемент как фотографию со свойствами времени съемки и устройства.</span><span class="sxs-lookup"><span data-stu-id="cdc57-116">Images taken with a camera (photos) have a [**photo facet**][photo] that identifies the item as a photo and provides the properties of when the photo was taken and with what device.</span></span>

<span data-ttu-id="cdc57-117">элементы с аспектом **folder** выполняют роль контейнеров элементов, поэтому у них есть ссылка `children`, указывающая на коллекцию объектов **driveItem** в папке.</span><span class="sxs-lookup"><span data-stu-id="cdc57-117">Items with the **folder** facet act as containers of items and therefore have a `children` reference pointing to a collection of **driveItems** under the folder.</span></span>

><span data-ttu-id="cdc57-118">**Примечание.** В OneDrive для бизнеса и библиотеках документов SharePoint свойство **cTag** не возвращается, если у ресурса **driveItem** есть аспект [folder][].</span><span class="sxs-lookup"><span data-stu-id="cdc57-118">**Note:** In OneDrive for Business or SharePoint document libraries, the **cTag** property is not returned, if the **driveItem** has a [folder][] facet.</span></span>
